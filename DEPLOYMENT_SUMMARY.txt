🚀 DÉPLOIEMENT RAPIDE (15 minutes)

Étape 1: Supabase (2 min)
────────────────────────
→ Allez sur supabase.com
→ Sign up + Create project
→ Copiez DATABASE_URL
→ Prêt!

Étape 2: Render (5 min)
──────────────────────
→ render.com
→ New Web Service
→ Connectez votre GitHub
→ Configuration:
  - Root: backend
  - Build: npm install && npm run build
  - Start: npm start
→ Ajoutez env vars:
  - DATABASE_URL (de Supabase)
  - JWT_SECRET (généré)
  - CORS_ORIGINS
  - NODE_ENV=production
→ Deploy!

Étape 3: Vercel (3 min)
──────────────────────
→ vercel.com
→ Import Project
→ Sélectionnez emploiplus
→ Import
→ Ajoutez env var:
   - VITE_API_BASE_URL = https://your-production-api.example.com
→ Deploy!

Étape 4: Tester (5 min)
──────────────────────
→ Ouvrez frontend: https://emploiplus.vercel.app
→ Tests des endpoints
→ Vérifiez logs

💡 POINTS CLÉ À RETENIR

1. Ne jamais commiter .env fichiers
2. JWT_SECRET doit être identique partout
3. DATABASE_URL provient de Supabase
4. CORS_ORIGINS doit contenir votre Vercel domain
5. Render redeploy auto quand env vars changent
6. Vercel redeploy auto à chaque git push
7. Logs sont vos meilleurs amis (Render/Vercel dashboards)
8. Supabase fait les backups auto
